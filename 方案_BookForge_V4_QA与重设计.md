# BookForge V4 · 整体 QA 与信息架构重设计

> 设计立场：V3 的问题不是缺说明，而是**界面没有自己解释自己**。本次按四条原则重做信息架构，而不是叠加引导层：
> ① Show, don't tell——过程本身可见，就不需要文字解释；② 内容即界面——屏幕最大区域留给真实内容，不留给装饰动效；③ 每屏一个主行动——用户永远知道"现在该干嘛"；④ 状态可直接操纵——能看到的状态就能拨动它，能拨动就能拨回来。

## 二、V4 信息架构

```
①扫描（默认屏，含初始态）   ②工坊            ③发布              ④增长
┌─────────┬───────┬─────────┐   审阅/修改/QA复审   开关式上架 + 手机实况   Q1赚多少/Q2健康吗/Q3怎么更好
│ 原文     │ 知识原子 │ 数字资产   │
│ 逐句扫描  │ 按章累积 │ 4张实时组装 │
│ 灰=归档   │ 飞入动画 │ +42批量计数 │
└─────────┴───────┴─────────┘
底部状态条：原文100%保留 · 已归档N段 · 原子N · 资产N
```

扫描屏三栏就是产品论点本身：**左栏证明"不丢内容"，中栏证明"可溯源"，右栏证明"能变钱"。**

## 三、扫描流水线规格

- 原文为《让大象飞》第 1 章真实课程文稿段落 + 第 2-6 章方法论摘句（对应全部 20 颗原子）；
- 节奏：段落滚入 → 扫描微光掠过 → 关键句琥珀高亮 0.5s → 句子飞至中栏结晶为原子条 → 原句留下划线+原子编号上标 → 段落其余文字变灰（计入"已归档原文库"）；
- 右栏 4 张样板资产（闪课①/自检清单模板/金句卡/F1 测评）按原子到位实时点亮槽位，成型后显示价格、可点进工坊；
- 扫描完成：其余 42 项资产以批量计数器生成（4→46），按 9 类显示分布——既诚实（不逐一表演）又表达规模化；
- 全程可「跳过 ≫」直达完成态；完成态出现双 CTA：打开工坊 / 直接上架。

## 四、发布开关规格

一行 = 资产：色点 + 标题 + 价格 + 开关。规则：开 = Approved（手机即时上架），关 = Internal（即时撤下，反悔零成本）；risk=High 的行开关锁定并显示「🔒 解锁需确认」，点击行内展开风险说明与「人工已确认」按钮，确认后开关解锁——Pending→Internal→Approved→（可回退）全链路在一行内完成。顶部保留「⚡ 一键上架演示集」与已上架计数。
