# BookForge V2 · 全流程设计与闭环分析

> V1 证明了"一本书能炸出 46 张卡"；V2 要证明的是**这条生产线能赚钱、能管控、能复利**。
> 视角：双视角——主线是操盘人控制台（编辑/运营/出版方），内嵌手机框 C 端预览证明终端体验。

---

## 一、V1 的三个断点，正是 V2 要补的全流程

V1 停在"卡片诞生"，留下三个无法回答的问题：

1. **卡片背后是什么？** 点开课程卡只能看 DNA 字段，看不到那门 4200 字的真课——而工程里《第1讲全层级演示》已经把闪课①、F1 测评 12 题、作业 Rubric、AI 教练点评实例全部生产出来了，这是最有说服力的弹药，V1 没用上；
2. **生产出来然后呢？** 没有修改、QA 复审、签核、上架的链路——"敢给法务看"停留在口号；
3. **凭什么值钱？** 没有转化率、收入、数据回流——《收费闭环》文档里 90 天 ¥98,700 的测算没有被可视化。

## 二、五幕全流程

```
① 投料 Ingest → ② 熔炼 Forge → ③ 工坊 Studio → ④ 发布 Publish → ⑤ 增长 Growth
       ↑                                ↑_____________数据回流（⑤的告警一键跳回③修内容）_____________|
       |_____________________________"丢下一本书"（⑤的终点 CTA 回到①）
```

### 第①幕 · 投料：用户怎么进来，能丢什么

入口是**书架 + 熔炉**双区：

- **书架**承载"已熔炼/排队中"的物料：《让大象飞》常驻（已熔炼，点击直达资产宇宙，**跳过演出**——给老板演示不等动画）；《人机沟通法则》排队位（呼应第二本复用验证）；空位引导投料。
- **投料不限于书**。凡承载作者方法论的内容都可熔炼：图书 PDF/EPUB、演讲/课程转写稿、专栏合集、播客转写、作者访谈、内部讲义。这把产品从"拆书工具"升维成"作者 IP 资产化引擎"——物料类型以可点选的 chips 呈现，拖任意文件进来自动识别。
- 演示模式下，新物料统一以《让大象飞》已验证流水线展演，并以 toast 声明。

### 第②幕 · 熔炼：压缩到 ~18 秒 + 可跳过

保留 V1 三幕剧（粒子→原子→卡片）但节奏加速；全程提供「跳过演出」。结束后 CTA 直指工坊："看这些原子如何变成一门能卖 ¥29.9 的课 →"。

### 第③幕 · 工坊：从原子到具体课程（V2 核心增量）

布局：左侧资产树（46 项按 8 类分组）+ 主区内容详情 + 右侧 QA 日志。

- **课程是真的**：闪课①以全文呈现（开场特斯拉案例、宝箱-钥匙、独角兽事实、25 年时机、自检清单五问、测验、作业、升级钩子）——内容逐段来自《第1讲全层级演示》文档；
- **每一段都带溯源**：段落左缘色条 🟦原书 / 🟨平台加工 / 🟩编辑原创，段头挂 source_atom 标签，点击标签回看原子内容——"原子 → 段落 → 课程"的生产链第一次肉眼可见；
- **每一段都可修改**：hover 出现编辑钮 → 行内编辑 → 保存后该段立刻进入「已修改 · QA 复审中」（橙色），约 2 秒后模拟 QA v2 通过转绿，右侧 QA 日志自动追加记录。这演示了核心管控原则：**任何人工修改都自动触发复审，内容永远处于受控状态**；
- **风险点是诚实的**：iPhone 引文段常驻 Medium 黄标"引文待与中信译本核对"——QA 不是装饰，是随处可见的事实；
- **深内容 + 批量证明**：闪课①/F1 测评/AI 教练点评/权益卡有全量内容；其余资产打开为标准 DNA 页并注明"由同一流水线按同模子产出（演示包收录第 1 讲垂直切片）"——既展示深度又暗示规模化。

### 第④幕 · 发布：状态机 + C 端镜像

- **三列看板**：Pending（高风险熔融态）→ Internal Demo Ready → Approved（法务签核，金边）。操盘人点「签核」卡片即流转——对应文档的 publish_level / legal_review_status 状态机；
- **发布即镜像**：旁边手机框是 C 端实况。**签核前手机里空空如也**（"暂无公开资产"），每签核一张，手机里实时多一个入口——"前台只能展示 Approved"这条规则从字段变成了体感；
- 手机内可真实体验：F1 测评试玩 3 题出画像（技术信徒→推荐闪课①）、闪课①移动端阅读（与工坊同一数据源，工坊改完这里同步变）、完课证书卡与导师 comment 卡样张；
- 合规红线常驻看板底部：不得出现"作者本人"、不承诺融资收益、传播物料需 Approved。

### 第⑤幕 · 增长：转化漏斗 + 数据回流

- **六层漏斗动画**（数据=《收费闭环》90 天测算，标注"演示测算"）：3000 测评 → 150 闪课(5%) → 30 直购(1%) → 30 升级(20%) → 20 行动营 → 3 Coffee Chat，**合计 ≈¥98,700，毛利 >85%**；
- **每层通过线仪表**：测评完成率>60%、闪课→中价≥15%、P3 占比>P2、定金≥20 人、完营率≥70%、NPS≥40——绿灯/红灯直观呈现"任何一条连续两次不达标就回上一层修产品"的纪律；
- **数据回流告警**（闭环的关键一跳）：如"第 3 讲完课率偏低 → 建议为 ka-301 补充本土案例"，每条告警带「去工坊修改」按钮一键跳回第③幕对应资产——**增长数据反哺内容生产，环就闭上了**；
- 终点 CTA：「把第二本书丢进来」回到第①幕。

## 三、四条闭环路径（产品价值的证明结构)

| 闭环 | 路径 | 在原型中的对应 |
|---|---|---|
| **经济闭环** | 免费测评引流 → ¥29.9 筛付费意愿 → ¥299/599 主力利润 → ¥2999 行动营 → 顶层服务 | 第⑤幕漏斗 + 第④幕价格阶梯 |
| **数据闭环** | 测评/作业/完课数据 → 定位讲不清的原子 → 回写原子库与课程 | 第⑤幕告警 →「去工坊修改」→ 第③幕编辑 → QA 复审 |
| **供给闭环** | 行动营优秀毕业生 → 下期助教 → 真人点评供给扩容 → 高价层可规模化 | 第⑤幕漏斗注释 + 行动营卡详情 |
| **版权闭环** | 内部 Demo（全 Pending）→ 收入数据 → 向作者/中信提案分成 → 授权 → 解锁作者本人权益 → 客单上探 | 第④幕状态机（一切从 Pending 开始）+ 第⑤幕收入测算即谈判筹码 |

一句话给观看者的价值叙事：**左手是一本书，右手是 90 天 ¥98,700 的测算和一条敢给法务看的生产线，中间每一步都摸得到、改得了、追得回。**

## 四、技术与数据

- 仍为单文件 HTML（prototype_BookForge_V2.html），零依赖，双击即开；五幕为 SPA 分屏，顶部进度导航；
- 数据三层：ATOMS（20 原子）/ CARDS（46 资产）/ DETAILS（闪课①全文分段、F1 十二题、Rubric、AI 点评实例、证书文案、漏斗与通过线）——全部取自工程文档，转化率为文档测算假设并明确标注；
- 工坊编辑与手机预览共用同一数据对象，改动即时双向呈现（为阶段 2 接 Next.js + JSON 的真实形态做同构预演）；
- 接真实流水线的接口不变：演出与资产均由事件流驱动，后续以 SSE 替换脚本即可。
