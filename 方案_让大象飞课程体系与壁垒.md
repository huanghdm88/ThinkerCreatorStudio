# 《让大象飞》课程体系推进方案（一人 + 一 AI 版）

> 目标：以课程资产为切入点，30 天内做出可上线的内部 Demo，90 天内完成付费验证，同时把壁垒建在"别人抄不走"的地方。
> 前提：不依赖团队、不开发后台、不接数据库。现有 Next.js demo（Thinker_Steve）即为上线载体。

---

## 一、为什么从"课程"切入，而不是全资产并行

Book-to-Assets 定义了 8 类资产，但一人启动时必须选一个能同时验证三件事的形态：

1. **付费意愿**：课程是客单价最高、用户最习惯付费的形态（观点卡免费、模板低价、活动重运营）；
2. **生产链路**：课程是对知识原子要求最完整的资产——概念、案例、工具、练习全要用到，跑通课程等于跑通了 80% 的原子库质量；
3. **数据飞轮**：课程有作业和完课行为，是唯一天然产生"用户学习数据"的资产，这是后面壁垒的原料。

观点卡和模板不放弃，但定位改为**课程的副产品**：每模块顺手产出 3-5 张观点卡（引流）和 1 个模板（交付物），不单独立项。

## 二、课程体系设计

### 2.1 产品分层（同一套原子库，三层变现）

| 层级 | 形态 | 价格定位 | 作用 |
|---|---|---|---|
| L0 | 观点卡 / 章节金句 / 免费第一讲 | 免费 | 引流、社交传播 |
| L1 | 6 模块微课（自学版） | 低客单（百元级） | 规模化收入、验证内容质量 |
| L2 | 21 天创业行动营（微课 + 作业 + AI 教练点评 + 社群） | 中客单（千元级） | 核心利润、产生学习数据 |
| L3 | 企业内训 / 工作坊（线下用 L2 素材重组） | 高客单 | 中信 B 端资源变现 |

启动顺序：L1 → L0（同步产出）→ L2 → L3。L3 不需要新内容，只需要重新打包。

### 2.2 课程大纲（沿用 demo 已验证的 6 模块结构）

1. 寻找方向——技术并没有你想的那么重要
2. 准备启航——用最少资源做最快验证
3. 打造产品——让用户爱上它
4. 锁定市场——先吃透用户
5. 增长引擎——从 0 到 1 之后的飞轮
6. 高速运转——组织、融资与持续创新

### 2.3 课程单元 Schema（核心：把 mock-data 里的结构固化为标准）

现有 mock-data.ts 中的章节结构已经是一个很好的课程单元 schema，直接固化：

```json
{
  "unit_id": "",
  "title": "",
  "source_atoms": ["ka-001", "ka-002"],
  "key_insights": [{ "icon": "", "title": "", "body": "", "content_source_type": "" }],
  "quote": { "text": "", "source": "", "author_claim_level": "" },
  "framework": { "name": "", "steps": [] },
  "case_studies": [{ "company": "", "lesson": "", "content_source_type": "" }],
  "homework": { "prompt": "", "submit_type": "", "rubric": [] },
  "derived_assets": { "cards": [], "template": "" },
  "qa": { "risk_level": "", "publish_level": "", "legal_review_status": "" }
}
```

关键改动有三处：

- 每个字段挂 `source_atoms`，强制课程内容溯源到知识原子，而不是 AI 自由发挥；
- `homework` 增加 `rubric`（评分维度），这是 AI 教练点评的依据；
- `derived_assets` 让观点卡和模板从课程单元自动派生，不另起炉灶。

### 2.4 课程的差异化：不是"讲书"，是"练方法"

每个单元固定五段式：**概念（书中观点）→ 案例（书中 + 补充）→ 工具（自检清单/模板）→ 练习（作业）→ 反馈（AI 教练点评）**。

前四段市面上的拆书课都有，第五段是差异化所在：

- **AI 创业教练**：基于知识原子库做 RAG 的点评 Agent。学员提交作业后，AI 按该单元的 rubric + 书中框架点评，引用原书观点时标注来源章节；
- 这把课程从"内容消费"变成"互动服务"，单位成本接近零，但体验是录播课给不了的；
- 实现成本极低：第一版就是一个带知识原子库上下文的 Claude/GPT prompt，放在课程页的输入框后面，或者干脆先用飞书/微信群人肉转发 AI 点评结果。

## 三、生产流水线（一人 + 一 AI，按周推进）

### W1：原子库与大纲定稿

- 用已验证的《让大象飞》Demo 包为底，补全 6 模块对应的知识原子（预计 40-60 个原子够用）；
- 固化 2.3 的课程单元 schema 和 4 个生产 Prompt（拆书 / 课程设计 / QA / 修复），存为项目内的 prompts/ 文件夹——这就是你的"生产线代码"；
- 产出：`02_知识原子库.json`（课程版）、课程大纲、schema 文档。

### W2：内容生产 + 双轮 QA

- 每天 1-2 个单元：AI 生成初稿 → QA v1（重点查误归因、content_source_type、商业承诺）→ 修复方案表 → 结构化修复 → QA v2；
- 同步派生每单元 3-5 张观点卡 + 1 个模板；
- 产出：`course_assets.json`（6 单元全量）+ 观点卡/模板 JSON。

### W3：上线内部 Demo

- 把 `course_assets.json` 放进 `/public/data/`，现有 `/course/[cardId]/learn` 页面从读 mock 改为读 JSON（这是唯一需要的代码改动，半天工作量）；
- 加一个最简作业提交 + AI 点评入口（第一版可以是"提交后跳转飞书表单，AI 点评由你人肉跑后回填"）；
- 部署 Vercel，生成内部链接；
- 产出：可发给老板/编辑/作者看的在线 Demo + Demo 包文档。

### W4：内测与数据

- 拉 10-20 个目标用户（创业者/产品人）免费走完 L2 流程的精简版（3 个单元 + 作业 + AI 点评）;
- 收集：完课率、作业提交率、AI 点评满意度、"愿意付多少钱"；
- 产出：内测报告 → 决定 L1 定价和是否推进付费。

### 付费验证（W5-W8，不写代码）

不开发支付系统。用现成渠道验证：知识星球 / 小鹅通 / 飞书付费群任选其一承载 L2 训练营，App 内 Demo 只做展示和体验入口。先验证"有人付钱"，再考虑产品化收款。

## 四、壁垒：建在哪里，怎么建

先说诚实的判断：**用 AI 把书变成课，这个动作本身零壁垒**，任何人拿 ChatGPT 一周也能做出"看起来差不多"的东西。壁垒只能建在以下五层，按硬度排序：

### 1. 授权壁垒（最硬，法律层面排他）

- 与作者/中信签订**"图书数字资产开发与运营授权"**——不是普通的电子版权，而是明确覆盖：课程化、卡片化、模板化、AI 训练与引用、Agent 知识库使用；
- 拿到授权后，别人做同样的事就是侵权。这是唯一别人**法律上不能复制**的壁垒；
- 一人启动阶段的做法：先做内部 Demo（全部资产标 Pending / Internal），用 Demo 去谈授权——**Demo 本身就是谈判筹码**，让作者和出版社看到"你的书在我们系统里长什么样"，比 PPT 提案有效得多；
- 升级形态：作者独家入驻 + 收益分成，作者越多，后来者越难撬。

### 2. 结构化资产壁垒（数据层面，越积越深)

- 别人有的是"内容"，你有的是**带溯源、带风险标注、带状态机的结构化资产**：knowledge_atom + content_source_type + author_claim_level + publish_level；
- 这意味着你的资产是**可审核、可交易、可进 App、可喂 Agent** 的，而别人的 AI 生成内容是一次性的、不可追责的；
- 跨书积累后形成**知识图谱**（《让大象飞》的"精益验证"原子可以和第二本书的原子互相引用），这个图谱每加一本书价值递增，复制成本递增；
- 一人阶段的做法：**严格执行 schema，绝不偷懒**。壁垒不在某个字段，而在"每一条资产都经得起法务和编辑检查"这件事坚持了 N 本书。

### 3. 流程/信任壁垒（出版社级 QA，C 端 AI 产品做不到）

- 双轮 QA + 修复方案表 + 人工签核的流水线，沉淀为**可复用的 SOP + Prompt 资产 + 评测集**；
- 这是对出版社的核心卖点："我们的 AI 生产线产出的内容，敢给法务看"。普通 AI 拆书产品根本过不了出版社的合规线，这条线就是护城河；
- 一人阶段的做法：每跑一本书，把 QA 发现的问题类型积累成**检查清单和自动化评测用例**。跑完 3 本书后，你的 QA Prompt 见过的坑是新进入者的几十倍。

### 4. 数据飞轮壁垒（时间层面，先发优势）

- 课程独有：学员作业、提问、卡点、完课数据回流 → 知道哪个原子讲不清、哪个案例最打动人、哪个作业没人交 → 反哺资产优化和下一本书的生产决策；
- AI 教练的点评质量随积累的"作业-点评-反馈"样本提升，这是后来者没有的训练数据；
- 一人阶段的做法：从内测第一天就把作业和反馈存成结构化数据（哪怕只是 JSON 文件），不要散落在微信聊天记录里。

### 5. 关系网络壁垒（最软但最难买）

- Hoffman / Founders Space 的作者关系、中信的版权库和 B 端渠道，是资金买不来的；
- 每成功运营一位作者，就是对下一位作者的最强案例；作者聚集形成平台网络效应。

### 不要把资源花在伪壁垒上

- **不要**指望技术架构形成壁垒（Next.js / RAG / 向量库人人都有）；
- **不要**过早做 CMS、后台、批量系统——在授权和付费验证之前，这些是成本不是壁垒；
- **不要**把 Prompt 当核心机密（会泄露、会过时），Prompt 的价值在于配套的评测集和 QA 数据，那个才留得住。

一句话总结壁垒结构：

> **授权决定别人能不能做，结构化资产决定别人做不做得像，QA 流程决定出版社信不信你，数据飞轮决定别人追不追得上。**

## 五、最小上线路径（零新基建）

```
已有 Next.js demo（专家页 + 课程页 + learn 页）
        │
W3: mock-data → /public/data/course_assets.json   ← 唯一代码改动
        │
Vercel 部署（内部链接，资产全部 Internal Demo Ready）
        │
作业提交：飞书表单 / 微信收集 → AI 点评（人肉中转）
        │
付费承载：知识星球 / 小鹅通（不自建支付）
```

上线判定标准（对齐项目文档第十一节）：内部 Demo 页只要求 `demo_status = Internal Demo Ready`；任何公开传播的观点卡/试听内容必须 `legal_review_status = Approved` 且不含长篇原文摘抄。

## 六、里程碑与验收

| 时间 | 里程碑 | 验收指标 |
|---|---|---|
| 30 天 | L1 课程 Demo 上线 + 内测 | 6 单元全量、双轮 QA 通过、10+ 内测用户、完课率 > 40%、作业提交率 > 30% |
| 60 天 | 第二本书复用验证 + 授权谈判启动 | Prompt/Schema 复用率 > 80%、用 Demo 完成一次对作者或中信的正式演示 |
| 90 天 | 付费验证 | L2 训练营首期收费开营（哪怕 20 人），或拿到一份数字资产授权意向 |

90 天后再决定是否投入 CMS、批量流水线和正式产品化——届时已有数据支撑决策，而不是凭信仰投入。

---

## 七、风险边界（一人阶段尤其要守住）

1. 所有课程内容默认 Pending，公开传播的物料（观点卡、试听）单独走法务/编辑确认；
2. AI 教练点评必须声明"基于《让大象飞》方法论的 AI 助教"，**绝不暗示作者本人参与**；
3. 案例补充（书外案例）必须标 `content_source_type = 平台加工`，与原书内容视觉上可区分；
4. 训练营不承诺融资、收益、成功率；
5. 内测阶段不收费、不公开，规避授权未定时的版权风险。
