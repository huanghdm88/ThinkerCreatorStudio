# BookForge · 「丢一本书进来」交互 Web 产品定义

> 定位：Book-to-Assets 系统的**前台体验层**。用户（老板 / 编辑 / 作者 / 中信）丢一本书进来，亲眼看着它被分解成知识原子，再不停孵化出 8 类数字资产卡片。
> 模式：演示与真实兼顾——第一版预生成数据 + 演出感编排，接口预留真实 LLM 流水线。

---

## 一、核心隐喻：知识熔炉（The Forge)

市面上所有"AI 拆书"产品的交互都是：上传 → 转圈等待 → 一次性吐出结果列表。**这中间的黑盒正是我们要打破的地方**——因为 Thinker 的核心壁垒恰恰在中间层（知识原子、溯源、QA）。所以交互隐喻设计为三幕剧：

```
第一幕 · 投书     书被拖入熔炉，封面碎裂为发光粒子流（每个粒子 = 一段原文）
第二幕 · 原子化   粒子凝聚成悬浮的「知识原子」星云，每颗原子可触摸、可查看溯源
第三幕 · 孵化     原子之间拉出光线、结晶成卡片，卡片源源不断飞向 8 条资产流
```

**差异化的本质：把生产线本身变成产品体验。** 别人展示结果，我们展示"一本书的数字生命如何诞生"——这同时就是给中信和作者的最强提案演示。

## 二、市面上没有的 5 个交互点

1. **物理投喂（Throw, not Upload）**：没有"上传按钮"。屏幕中央是一座环形熔炉，用户把书（文件或书架上的书封）**拖拽甩进去**，松手瞬间书脊裂开、纸页粒子化被吸入。支持真实拖入 PDF/EPUB/MD 文件。
2. **可触摸的中间层（Living Atoms）**：原子不是进度条，是真实悬浮的发光节点。鼠标悬停看到原子内容（ka-101 技术陷阱…）；**点一颗原子，所有由它派生的卡片同时亮起并拉出溯源光线**——知识图谱第一次变成可玩的东西。
3. **永不停止的孵化流（Endless Hatch）**：卡片不是一次性铺满，而是像爆米花一样持续弹出，按 8 类资产分流。营造"这本书还在继续产出"的感觉——对应项目愿景"每本书拥有持续生长的数字生命"。
4. **卡片有温度（QA 状态可视化）**：新生卡片是"熔融态"（橙红光晕，Pending），经过 QA 光环后"冷却"为蓝色（Internal Demo Ready），法务确认后变金边（Approved）。**风险状态不是后台字段，是用户看得见的物理状态**——这是"敢给法务看"的可视化表达。
5. **翻卡溯源（Flip to Source）**：任何卡片点击翻面，背面是它的 DNA：source_atoms、content_source_type（🟦原书 / 🟨平台加工 / 🟩编辑原创）、risk_level、publish_level。溯源即卖点。

## 三、卡片体系（与文档对齐，共 8 大类 + 权益类）

| # | 资产类 | 子类 | 卡面主色 |
|---|---|---|---|
| 1 | 课程卡 Course | 微课 / 训练营 / 闪课 / 章节精讲 / 企业内训 | 靛蓝 |
| 2 | 模板卡 Template | 自检表 / 评分表 / 工作表 / 访谈表 / 提示词模板 | 青绿 |
| 3 | 观点卡 Insight | 金句 / 观点 / 行动建议 / 反直觉 / 案例启发 | 琥珀 |
| 4 | 活动卡 Event | 读书会 / 沙龙 / 工作坊 / 黑客松 / 闭门会 | 玫红 |
| 5 | AMA 卡 | 开放问答 / 专题讨论 / 行业议题 | 紫 |
| 6 | Coffee Chat 卡 | 1v1 诊断 / 复盘 / 咨询 | 棕橙 |
| 7 | 导师计划卡 Mentorship | 30 天计划 / 90 天训练营 / 企业陪跑 | 深蓝 |
| 8 | Agent 资产卡 | 专家 Agent 知识库 / 章节问答 / API 资产 | 银白 |
| + | 权益卡（衍生） | 测评卡 / 证书卡 / 徽章 / 导师 comment 卡 | 金 |

每张卡片统一 schema（对齐 Thinker2.0 第八节）：`asset_id, type, title, summary, source_atoms[], content_source_type, risk_level, publish_level, legal_review_status, price_tier`。

## 四、页面结构

```
/            熔炉主舞台（投书 → 原子 → 孵化，全屏沉浸）
/forge/:bookId   某本书的资产宇宙（原子星云 + 8 条资产流 + 溯源图谱）
/card/:assetId   卡片详情（正面内容 + 背面 DNA + QA 历史）
/shelf       书架（已熔炼的书 = 已验证案例；《让大象飞》常驻）
/admin       （阶段 3）状态控制：Pending → Internal → Approved 签核
```

主舞台布局：中央熔炉占 50%，顶部实时计数器（原子数 / 卡片数 / 各类分布 / 风险分布），两侧 8 条资产流瀑布。

## 五、演出编排（预生成数据版的"实时感"）

数据源：《让大象飞》Demo 包（ka-101~108 等原子、闪课①-⑥、F1 测评、P2-P5 矩阵、证书/comment 卡文案——全部来自已有文档，零编造）。

时间线脚本（总长约 60-90 秒，之后进入无限慢速孵化）：

| 时间 | 事件 |
|---|---|
| 0-3s | 书被吸入，封面碎裂，2000+ 粒子涌入熔炉 |
| 3-8s | 粒子分章聚拢，逐颗点亮知识原子（带 ID 和标题浮现） |
| 8s+ | 原子间随机拉线 → 结晶 → 卡片弹出，节奏由密到疏 |
| 15s | 第一张课程卡诞生时全屏微震 + 计数器跳动 |
| 30s | QA 光环扫过，一批卡片由熔融转冷却色 |
| 60s+ | 进入余韵模式：每 5-10 秒仍有新卡孵出，永不停止 |

真实模式接口预留：演出引擎只消费一个事件流 `{type: 'atom'|'card'|'qa', payload}`。预生成版本由脚本定时发事件；真实版本改为 SSE 接 LLM 流水线输出，**前端零改动**。

## 六、技术方案

- **阶段 1（本原型）**：单文件 HTML，Canvas 粒子引擎 + DOM 卡片层 + CSS 3D 翻卡，数据内嵌 JSON。零依赖、可直接浏览器打开、可投屏演示。
- **阶段 2**：并入现有 Next.js demo（Thinker_Steve），数据移至 `/public/data/app_assets_demo.json`，路由按第四节落位。真实拖入文件 → 调用 Book-to-Assets Agent 流水线（Claude）→ SSE 推流驱动演出。
- **阶段 3**：Supabase 存资产 + 状态机，/admin 签核台，QA 报告挂卡片背面。
- 视觉基调：深空黑底 + 发光粒子（参考 Linear/Vercel 暗色美学），熔炉用径向渐变 + 噪声动画，避免廉价 3D。

## 七、为什么这个交互本身是壁垒叙事

演示时的一句话：**"别人给你看 AI 生成的结果，我们给你看一本书的数字生命如何诞生——每一张卡都能摸到它来自哪个原子、哪一章、什么风险等级。"** 可触摸的原子层 = 结构化资产壁垒的可视化；卡片温度 = QA 流程壁垒的可视化；无限孵化 = "可复利知识资产"愿景的可视化。交互设计与商业叙事完全同构。
